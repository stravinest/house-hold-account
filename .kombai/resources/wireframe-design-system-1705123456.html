4. 접근성 개선

항목 현재 문제 개선안 기준
터치 타겟 일부 아이콘 버튼 작음 최소 44x44dp 보장 Material Design
색상 대비 일부 회색 텍스트 대비 낮음 최소 4.5:1 보장 WCAG AA
포커스 표시 키보드 포커스 불명확 명확한 outline 추가 WCAG 2.4.7
시맨틱 라벨 일부 아이콘만 있는 버튼 semanticLabel 추가 스크린리더 지원

5. 다크 모드 일관성

❌ 현재 문제

하드코딩: Colors.grey[400]

하드코딩: Colors.grey[600]

하드코딩: Colors.red

→ 다크 모드에서 대비 문제 발생

✓ 개선안

ColorScheme.onSurfaceVariant

ColorScheme.onSurface.withOpacity(0.6)

ColorScheme.error

→ 테마에 따라 자동 조정

7. 구현 우선순위

우선순위 작업 영향도 예상 시간
P0 (Critical) 디자인 토큰 파일 생성 (design_tokens.dart) 전체 앱 2시간
P0 하드코딩 색상 → ColorScheme 전환 다크 모드 일관성 4시간
P1 (High) 공통 위젯 생성 (AppCard, SectionHeader, EmptyState) 코드 중복 제거 3시간
P1 borderRadius 통일 (8→12) 시각적 일관성 1시간
P2 (Medium) 타이포그래피 통일 (textTheme 사용) 가독성 2시간
P2 접근성 개선 (터치 타겟, semanticLabel) 사용성 3시간
